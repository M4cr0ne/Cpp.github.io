LAMBDA

1. Idea chiave

Una lambda è, in sostanza, una funzione scritta “al volo”, proprio dove ti serve. Invece di dichiarare una funzione separata sopra o sotto il main(), puoi scrivere direttamente una piccola funzione all'interno del codice.

Questo è utile quando la funzione è breve, semplice e non verrà riutilizzata in molti punti diversi del programma.

In pratica: una lambda è una mini-funzione locale, molto comoda per azioni veloci e ben circoscritte.

2. Modello mentale

Immagina di avere bisogno di una piccola regola, per esempio “somma due numeri” o “controlla se un numero è pari”, ma solo in un punto del programma. Invece di costruire una funzione completa con nome, puoi usare una lambda.

3. Sintassi essenziale

La forma generale di una lambda è questa:

[capture](parametri) {
    corpo
}

Esempio base

auto somma = [](int a, int b) {
    return a + b;
};
capture []

Indica quali variabili esterne la lambda può usare.

auto

Spesso si usa auto per lasciare al compilatore il tipo della lambda.

4. Esempi guidati

Esempio 1 • Lambda che somma due numeri

#include <iostream>
using namespace std;

int main(){
    auto somma = [](int a, int b) {
        return a + b;
    };

    cout << somma(4, 6) << endl;
}

La lambda viene salvata nella variabile somma e poi richiamata come una normale funzione.

Esempio 2 • Lambda che controlla se un numero è pari

auto pari = [](int n) {
    return n % 2 == 0;
};

Qui la lambda restituisce un valore booleano: true se il numero è pari, false altrimenti.

Esempio 3 • Lambda usata subito

cout << [](int x) {
    return x * x;
}(5) << endl;

In questo caso la lambda viene definita e chiamata immediatamente, senza essere assegnata a una variabile.

Esempio 4 • Caso realistico: aggiungere IVA

auto aggiungiIVA = [](float prezzo) {
    return prezzo * 1.22;
};
Le lambda sono utili quando la logica è breve, chiara e non ha senso essere separata in una funzione con nome esterno.

5. Capture e uso locale

La parte tra parentesi quadre indica la capture, cioè quali variabili esterne la lambda può usare. Per una terza, la cosa più importante da ricordare è questa:

Capture Significato Esempio
[] Non usa variabili esterne [] (int x) { return x*x; }
[a] Usa la variabile esterna a [a](int x){ return x+a; }

Esempio con capture

int bonus = 10;

auto aggiungiBonus = [bonus](int prezzo) {
    return prezzo + bonus;
};
Per un livello iniziale, va benissimo concentrarsi soprattutto sulle lambda con [] vuoto, che sono più facili da capire.

6. Lambda vs funzione normale

Una lambda non sostituisce sempre una funzione normale. Serve soprattutto per casi brevi e locali.

Funzione normale

Meglio quando la logica è lunga, importante o riutilizzata più volte.

Lambda

Meglio quando la logica è breve e serve solo in quel punto del codice.

Regola pratica: se una funzione è piccola e temporanea, la lambda può essere una scelta elegante. Se è centrale nel programma, meglio una funzione normale.

7. Errori tipici degli studenti

Errore 1 • Pensare che una lambda sia qualcosa di totalmente diverso da una funzione

In realtà è sempre una funzione, solo scritta in forma anonima e locale.

Errore 2 • Dimenticare parentesi quadre, tonde o graffe

La sintassi della lambda è più compatta, quindi è facile dimenticare un simbolo.

Errore 3 • Usare lambda troppo complesse

Se la logica diventa lunga, è meglio una funzione normale con nome.

Errore 4 • Non capire la capture

La parte tra quadre serve a dire quali variabili esterne possono essere usate dalla lambda.

Errore 5 • Pensare che auto sia opzionale solo per stile

Spesso è il modo più comodo per memorizzare una lambda, perché il suo tipo reale è complesso.

10. Riepilogo finale

  • La lambda è una funzione anonima.
  • Si scrive direttamente nel punto in cui serve.
  • È utile per operazioni brevi e locali.
  • Può avere parametri, return e corpo come una funzione normale.
  • La parte tra quadre indica la capture.
  • Spesso si assegna a una variabile con auto.
