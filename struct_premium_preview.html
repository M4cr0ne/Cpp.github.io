1. Idea chiave

Una struct serve quando una sola variabile non basta più. Se vuoi rappresentare uno studente, per esempio, non ti serve solo il nome oppure solo l'età: ti servono più dati insieme.

La struct permette proprio questo: creare un nuovo tipo di dato composto da più campi, ognuno con il proprio nome e il proprio tipo.

In pratica: una struct è come una scheda con più caselle etichettate, tutte riferite allo stesso oggetto o alla stessa persona.

3. Sintassi essenziale

La forma generale di una struct è questa:

struct NomeStruct {
    tipo campo1;
    tipo campo2;
    tipo campo3;
};

Esempio base

#include <iostream>
#include <string>
using namespace std;

struct Studente {
    string nome;
    int eta;
    float media;
};

int main(){
    Studente s1;
}

4. Esempi guidati

Esempio 1 • Assegnare valori ai campi

#include <iostream>
#include <string>
using namespace std;

struct Studente {
    string nome;
    int eta;
};

int main(){
    Studente s1;
    s1.nome = "Luca";
    s1.eta = 16;
}

I campi si raggiungono con l'operatore punto. Ogni campo si comporta come una normale variabile interna all'oggetto.

Esempio 2 • Stampare i dati

#include <iostream>
#include <string>
using namespace std;

struct Studente {
    string nome;
    int eta;
    float media;
};

int main(){
    Studente s1;
    s1.nome = "Anna";
    s1.eta = 17;
    s1.media = 8.5;

    cout << s1.nome << endl;
    cout << s1.eta << endl;
    cout << s1.media << endl;
}

Esempio 3 • Inizializzazione diretta

#include <string>
using namespace std;

struct Libro {
    string titolo;
    string autore;
    int anno;
};

Libro l1 = {"1984", "Orwell", 1949};

In questo caso i valori vengono inseriti subito nell'ordine dei campi dichiarati.

Esempio 4 • Caso realistico: scheda prodotto

#include <iostream>
#include <string>
using namespace std;

struct Prodotto {
    string nome;
    float prezzo;
    int quantita;
};

int main(){
    Prodotto p1;

    p1.nome = "Notebook";
    p1.prezzo = 3.50;
    p1.quantita = 20;

    cout << "Prodotto: " << p1.nome << endl;
    cout << "Prezzo: " << p1.prezzo << endl;
    cout << "Quantita: " << p1.quantita << endl;
}
La struct è utilissima quando più informazioni appartengono allo stesso oggetto e vuoi evitare variabili sparse e scollegate.

6. Struct vs Class

In C++ struct e class sono molto simili. A livello iniziale, la differenza più importante è che nella struct i membri sono pubblici di default, mentre nella class sono privati di default.

Aspetto Struct Class
Accesso predefinito public private
Uso didattico iniziale Ottima per dati semplici Più adatta a progettazione completa
Somiglianza Molto alta Molto alta

Quando è perfetta la struct

Quando devi solo raggruppare dati collegati in modo semplice e leggibile.

Quando entrerà in gioco la class

Quando oltre ai dati vorrai controllare meglio accesso, metodi e comportamento dell'oggetto.

7. Errori tipici degli studenti

Errore 1 • Dimenticare il punto

Non si scrive s1nome ma s1.nome.

Errore 2 • Confondere il nome della struct con il nome della variabile

Studente è il tipo. s1 è una variabile di quel tipo.

Errore 3 • Dimenticare il punto e virgola dopo la definizione

struct Studente {
    string nome;
    int eta;
}; // il punto e virgola finale serve

Errore 4 • Pensare che i campi siano variabili separate e indipendenti

I campi appartengono tutti allo stesso oggetto: descrivono insieme la stessa entità.

Errore 5 • Usare troppi dati scollegati invece di una struct

Se nome, età e media appartengono allo stesso studente, tenerli in variabili separate rende il programma più confuso.

10. Riepilogo finale

  • La struct crea un nuovo tipo di dato composto.
  • Serve per raccogliere più campi collegati tra loro.
  • I campi possono avere tipi diversi.
  • Si accede ai campi con l'operatore punto.
  • È ideale per rappresentare persone, prodotti, libri, studenti e oggetti simili.
  • Aiuta a scrivere programmi più ordinati e leggibili.
Se uno studente capisce bene la struct, fa un passo importante verso la programmazione orientata agli oggetti e verso una gestione più seria dei dati complessi.
