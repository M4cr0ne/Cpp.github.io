1. Idea chiave

Una class serve per modellare un oggetto del mondo reale o logico. Per esempio: uno studente, un'auto, un prodotto, un conto bancario. Ogni oggetto ha dati che lo descrivono e azioni che può compiere.

La class permette di tenere insieme queste due cose: le informazioni dell'oggetto e le funzioni che lavorano su quelle informazioni.

In pratica: la class è un progetto, mentre l'oggetto è una sua realizzazione concreta.

2. Modello mentale

Immagina una class come il modello di una carta d'identità intelligente: non contiene solo campi come nome ed età, ma anche azioni come mostrare i dati o modificare lo stato dell'oggetto.

3. Sintassi essenziale

La forma generale di una class è questa:

class NomeClasse {
private:
    dati_privati;

public:
    metodi_pubblici;
};

Esempio base

#include <iostream>
#include <string>
using namespace std;

class Studente {
private:
    string nome;
    int eta;

public:
    void setDati(string n, int e) {
        nome = n;
        eta = e;
    }

    void mostraDati() {
        cout << nome << " " << eta << endl;
    }
};
Private

I dati interni vengono protetti e non si modificano direttamente dall'esterno.

Public

I metodi pubblici sono il modo corretto per usare l'oggetto.

4. Esempi guidati

Esempio 1 • Creare un oggetto e usare i metodi

#include <iostream>
#include <string>
using namespace std;

class Studente {
private:
    string nome;
    int eta;

public:
    void setDati(string n, int e) {
        nome = n;
        eta = e;
    }

    void mostraDati() {
        cout << nome << ", " << eta << endl;
    }
};

int main(){
    Studente s1;
    s1.setDati("Anna", 17);
    s1.mostraDati();
}

Qui l'oggetto s1 viene creato dalla class Studente. I dati interni non vengono toccati direttamente: si usano metodi pubblici.

Esempio 2 • Metodo che modifica lo stato

class Contatore {
private:
    int valore;

public:
    void setValore(int v) {
        valore = v;
    }

    void incrementa() {
        valore++;
    }

    int getValore() {
        return valore;
    }
};

Una class non si limita a contenere dati: può anche definire il comportamento dell'oggetto, cioè ciò che l'oggetto sa fare.

Esempio 3 • Caso realistico: prodotto con sconto

#include <iostream>
#include <string>
using namespace std;

class Prodotto {
private:
    string nome;
    float prezzo;

public:
    void setDati(string n, float p) {
        nome = n;
        prezzo = p;
    }

    void applicaSconto(float percentuale) {
        prezzo = prezzo - prezzo * percentuale / 100;
    }

    void mostra() {
        cout << nome << " - " << prezzo << endl;
    }
};
La class è potente perché i dati non vengono sparsi nel programma: restano dentro l'oggetto e sono manipolati in modo controllato dai metodi.

5. Public e Private

Questo è uno dei punti più importanti della class. In una class, i membri sono private di default. Questo significa che dall'esterno non puoi toccarli liberamente, a meno che tu li renda pubblici o fornisca metodi per gestirli.

Sezione Significato Uso tipico
private: Dati protetti, non accessibili direttamente dall'esterno attributi interni come nome, età, prezzo
public: Parte accessibile dall'esterno metodi come set, get, mostra, modifica
L'idea di fondo è questa: non lasciare che chiunque modifichi i dati in qualsiasi modo, ma passare da metodi controllati.

6. Class vs Struct

Struct e class sono molto simili in C++. La differenza più evidente per chi inizia è il livello di accesso predefinito.

Aspetto Struct Class
Accesso predefinito public private
Uso iniziale Raggruppare dati Raggruppare dati + comportamenti
Idea didattica Più semplice Più vicina alla OOP

Quando basta una struct

Quando devi solo rappresentare dati collegati in modo semplice e diretto.

Quando conviene una class

Quando vuoi proteggere meglio i dati e definire azioni precise che l'oggetto può compiere.

7. Errori tipici degli studenti

Errore 1 • Pensare che class e oggetto siano la stessa cosa

La class è il modello. L'oggetto è un'istanza concreta creata da quel modello.

Errore 2 • Dimenticare il punto e virgola finale

class Prova {
    // contenuto
};

Errore 3 • Cercare di accedere direttamente ai dati private

Se un attributo è private, dall'esterno non puoi fare oggetto.nome.

Errore 4 • Confondere dati e metodi

Gli attributi descrivono lo stato. I metodi definiscono cosa l'oggetto può fare.

Errore 5 • Usare una class senza capire perché

La class ha senso quando ti aiuta a organizzare meglio il programma, non solo per complicare il codice.
