VECTOR

Il vector è un contenitore dinamico della STL che permette di memorizzare più elementi dello stesso tipo in una struttura ordinata, accessibile per indice e capace di crescere o ridursi durante l'esecuzione del programma.

1. Idea chiave

Un vector serve per gestire una collezione di dati dello stesso tipo quando non sappiamo in anticipo quanti elementi avremo, oppure quando questo numero può cambiare nel tempo.

Con un array classico bisogna decidere subito la dimensione. Con un vector, invece, possiamo iniziare vuoti e poi aggiungere elementi via via con push_back().

In pratica: il vector è come un array più comodo e più flessibile, ma leggermente più complesso da capire all'inizio.

2. Modello mentale

Immagina un vector come una fila di caselle numerate. Ogni casella contiene un valore. La differenza rispetto a un array è che il vector può allungarsi o accorciarsi mentre il programma gira.

Esempio: vector<int> numeri = {10, 20, 30, 40};

3. Sintassi essenziale

Per usare i vector servono la libreria <vector> e il namespace standard.

#include <iostream>
#include <vector>
using namespace std;

vector<int> numeri;         // vector vuoto di interi
vector<int> voti = {28, 30, 25};

4. Esempi guidati

Esempio 1 • Inserire elementi con push_back()

#include <iostream>
#include <vector>
using namespace std;

int main(){
    vector<int> numeri;

    numeri.push_back(10);
    numeri.push_back(20);
    numeri.push_back(30);

    cout << numeri[0] << endl;
    cout << numeri[1] << endl;
    cout << numeri[2] << endl;
}

Qui il vector parte vuoto. Ogni chiamata a push_back() aggiunge un elemento in fondo. Alla fine il vector contiene 10, 20, 30.

Esempio 2 • Leggere e stampare con un ciclo

#include <iostream>
#include <vector>
using namespace std;

int main(){
    vector<int> numeri;
    int valore;

    for(int i = 0; i < 5; i++){
        cout << "Inserisci un numero: ";
        cin >> valore;
        numeri.push_back(valore);
    }

    for(int i = 0; i < numeri.size(); i++){
        cout << numeri[i] << endl;
    }
}
Nota importante: usare numeri.size() nel ciclo è meglio che scrivere un numero fisso, perché il vector può avere una dimensione variabile.

Esempio 3 • Caso realistico: media dei voti

#include <iostream>
#include <vector>
using namespace std;

int main(){
    vector<int> voti;
    int n, voto, somma = 0;

    cout << "Quanti voti vuoi inserire? ";
    cin >> n;

    for(int i = 0; i < n; i++){
        cout << "Inserisci voto: ";
        cin >> voto;
        voti.push_back(voto);
        somma += voto;
    }

    cout << "Media = " << (float)somma / voti.size() << endl;
}

Questo esempio mostra bene perché il vector è utile: non sappiamo prima quanti voti inserirà l'utente.

6. Array vs Vector

Gli studenti spesso confondono array e vector perché entrambi permettono l'accesso per indice. Però non sono la stessa cosa.

Aspetto Array Vector
Dimensione Fissa Variabile
Aggiunta elementi No, non facilmente Sì, con push_back()
Libreria necessaria No Sì, <vector>
Comodità Più semplice Più flessibile
Uso tipico Quando sai già quanti dati hai Quando il numero di dati può cambiare
Regola pratica: se la quantità di dati è incerta o cresce durante il programma, il vector è quasi sempre la scelta più naturale.

7. Errori tipici degli studenti

Errore 1 • Dimenticare #include <vector>

Il compilatore non riconosce il tipo vector.

Errore 2 • Usare un indice che non esiste

Esempio sbagliato: v[5] quando il vector ha solo 3 elementi.

Gli indici validi vanno da 0 a size()-1.

Errore 3 • Confondere size() con l'ultimo indice

Se v.size() vale 4, gli indici sono 0,1,2,3. L'ultimo indice non è 4.

Errore 4 • Pensare che il vector abbia dimensione fissa come un array

Il vector può crescere. È proprio questa la sua utilità principale.

Errore 5 • Scrivere il ciclo con un limite sbagliato

// SBAGLIATO
for(int i = 0; i <= v.size(); i++)

// CORRETTO
for(int i = 0; i < v.size(); i++)

Con <= provi ad accedere anche a una posizione che non esiste.

10. Riepilogo finale

  • Il vector è un contenitore dinamico della STL.
  • Contiene elementi tutti dello stesso tipo.
  • Gli elementi sono ordinati e accessibili tramite indice.
  • Può crescere con push_back() e ridursi con pop_back().
  • size() restituisce il numero di elementi, non l'ultimo indice.
  • È più flessibile dell'array quando la quantità di dati non è nota in anticipo.
Se uno studente capisce bene array e vector, ha già costruito una base molto forte per affrontare le altre strutture dati del C++.
