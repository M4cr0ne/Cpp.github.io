1. Idea chiave

Una map serve quando vuoi associare un dato a una chiave. Per esempio: il nome di uno studente al suo voto, il nome di un prodotto al suo prezzo, oppure il codice di un paese alla sua capitale.

In un array o in un vector si accede usando un indice numerico. In una map, invece, si accede tramite una chiave, che può essere più significativa e più naturale da usare.

In pratica: la map funziona come un piccolo dizionario, dove a ogni parola-chiave corrisponde un valore.

2. Modello mentale

Immagina una map come una rubrica o un glossario. A sinistra c'è la chiave, a destra c'è il valore associato. La chiave deve essere unica, altrimenti ci sarebbe ambiguità.

Esempio: map<string, int> voti;

3. Sintassi essenziale

Per usare la map serve la libreria <map>.

#include <iostream>
#include <map>
#include <string>
using namespace std;

map<string, int> voti;
Forma generale

map<tipoChiave, tipoValore> nome;

Esempio: map<string, float> prezzi;

Chiave unica

Ogni chiave può comparire una sola volta.

Se riusi la stessa chiave, il valore viene aggiornato.

4. Esempi guidati

Esempio 1 • Inserire coppie chiave-valore

#include <iostream>
#include <map>
#include <string>
using namespace std;

int main(){
    map<string, int> voti;

    voti["Anna"] = 28;
    voti["Luca"] = 30;
}

Qui stiamo creando due associazioni: Anna → 28 e Luca → 30.

Esempio 2 • Leggere un valore tramite la chiave

cout << voti["Anna"] << endl;

Non stiamo usando un indice numerico, ma una chiave di tipo string.

Esempio 3 • Scorrere una map

for(auto elemento : voti){
    cout << elemento.first << " : " << elemento.second << endl;
}

In una map, first rappresenta la chiave e second rappresenta il valore.

Esempio 4 • Caso realistico: listino prezzi

map<string, float> prezzi;

prezzi["PLA"] = 19.99;
prezzi["PETG"] = 24.50;
prezzi["ABS"] = 27.00;
La map è perfetta quando il valore va cercato tramite un nome, un codice o un'etichetta unica.

6. Map vs Vector

Gli studenti spesso confondono map e vector perché entrambi contengono più dati. Ma la logica è diversa.

Vector

Usa un indice numerico.

Esempio: v[0], v[1], v[2]

Map

Usa una chiave significativa.

Esempio: m["Anna"], m["PLA"]

Regola pratica: se vuoi cercare un dato tramite nome, codice o etichetta, la map è spesso più adatta del vector.

7. Errori tipici degli studenti

Errore 1 • Pensare che la map usi indici numerici come il vector

No: la map si basa su chiavi, non su posizioni numerate.

Errore 2 • Dimenticare che la chiave deve essere unica

Se inserisci di nuovo la stessa chiave, il valore precedente viene sostituito.

Errore 3 • Non capire first e second

Quando scorri la map, first è la chiave e second è il valore.

Errore 4 • Usare una map quando bastava un vector

Se i dati vanno letti in ordine con indici numerici, un vector può essere più naturale.

Errore 5 • Usare m[chiave] senza pensare agli effetti

Accedere con parentesi quadre può creare automaticamente una nuova chiave se non esiste già.

10. Riepilogo finale

  • La map memorizza coppie chiave-valore.
  • Si usa quando vuoi cercare un valore tramite una chiave significativa.
  • Le chiavi devono essere uniche.
  • Non si lavora con indici come in array o vector.
  • Quando scorri una map, first è la chiave e second è il valore.
  • È molto utile per dizionari, listini, rubriche, stati e associazioni simili.
Se uno studente capisce bene la map, inizia a ragionare non più solo per posizione, ma anche per associazione logica tra dati.
