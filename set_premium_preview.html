SET

Il set è un contenitore STL che memorizza valori unici. Serve quando vuoi conservare elementi senza duplicati e verificare facilmente se un valore è presente oppure no.

1. Idea chiave

Un set serve quando vuoi memorizzare valori senza ripetizioni. Se provi a inserire due volte lo stesso elemento, il set ne conserverà una sola copia.

È molto utile, per esempio, per tenere traccia di codici unici, numeri già visti, categorie presenti o nomi distinti.

In pratica: il set funziona come un insieme matematico, dove un elemento può esserci oppure no, ma non può comparire due volte.

2. Modello mentale

Immagina di raccogliere numeri in una scatola speciale. Ogni volta che inserisci un numero già presente, la scatola lo rifiuta perché lo considera duplicato.

Esempio: inserimento di 5, 8, 5, 3
5
8
5
3

Risultato nel set: 3, 5, 8

1
Inserisci un valore

Se non c'è già, viene aggiunto.

2
Controllo automatico

Il set verifica se il valore è già presente.

3
Niente duplicati

Ogni valore compare al massimo una volta.

3. Sintassi essenziale

Per usare il set serve la libreria <set>.

#include <iostream>
#include <set>
using namespace std;

set<int> numeri;
Forma generale

set<tipo> nome;

Esempio: set<string> nomi;

Valori unici

Il set conserva solo elementi distinti.

Un secondo inserimento uguale non crea un duplicato.

4. Esempi guidati

Esempio 1 • Inserire valori

#include <iostream>
#include <set>
using namespace std;

int main(){
    set<int> numeri;

    numeri.insert(5);
    numeri.insert(8);
    numeri.insert(5);
}

Anche se il numero 5 viene inserito due volte, nel set resterà una sola volta.

Esempio 2 • Scorrere un set

for(int n : numeri){
    cout << n << endl;
}

Il set si può scorrere con un ciclo range-based. Non si usa l'indice come nei vector.

Esempio 3 • Verificare se un valore esiste

if(numeri.count(8)){
    cout << "Presente" << endl;
}

count() restituisce 1 se il valore è presente, 0 se non c'è.

Esempio 4 • Caso realistico: classi già visitate

set<string> classi;

classi.insert("3A");
classi.insert("4B");
classi.insert("3A");
Il set è perfetto quando vuoi sapere quali elementi distinti sono presenti, senza interessarti a quante volte sono stati inseriti.

6. Set vs Vector

Set e vector contengono entrambi più dati, ma non hanno lo stesso scopo.

Vector

Permette duplicati e usa indici.

È utile quando contano ordine e posizione.

Set

Non permette duplicati e non usa indici.

È utile quando conta solo la presenza distinta dei valori.

Regola pratica: se vuoi evitare ripetizioni in automatico, il set è spesso più adatto del vector.

7. Errori tipici degli studenti

Errore 1 • Pensare che il set conservi i duplicati

No: il set mantiene solo valori unici.

Errore 2 • Cercare di usare un indice

Non puoi fare s[0] come in un vector.

Errore 3 • Confondere presenza e frequenza

Il set dice se un valore c'è oppure no, non quante volte è stato inserito.

Errore 4 • Usare il set quando servono i duplicati

Se devi conservare tutte le ripetizioni, il set non è la scelta giusta.

Errore 5 • Dimenticare che conta la distinzione dei valori

Il senso del set non è l'ordine numerico, ma il fatto che ogni elemento sia unico.

10. Riepilogo finale

  • Il set contiene valori unici.
  • Non permette duplicati.
  • Non si usa con indici come array o vector.
  • È utile per verificare la presenza di un valore.
  • Funziona bene quando conta l'insieme distinto degli elementi.
  • È adatto a codici, categorie, numeri già visti e casi simili.
Se uno studente capisce bene il set, inizia a distinguere chiaramente tra “lista di elementi” e “insieme di valori unici”, che è un passaggio logico molto importante.
