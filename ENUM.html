ENUM

1. Idea chiave

Un enum si usa quando una variabile può assumere solo alcuni valori precisi e prestabiliti. Invece di usare numeri come 0, 1, 2 senza spiegazione, puoi dare loro nomi significativi.

Per esempio, se vuoi rappresentare i giorni della settimana, i livelli di difficoltà o lo stato di un ordine, un enum è molto più leggibile di una semplice variabile intera.

In pratica: un enum sostituisce numeri anonimi con etichette chiare che spiegano subito cosa rappresentano.

2. Modello mentale

Immagina un enum come una lista chiusa di etichette. Ogni etichetta ha un nome e dietro le quinte corrisponde a un numero intero.

Esempio: enum Giorno {LUNEDI, MARTEDI, MERCOLEDI, GIOVEDI};

3. Sintassi essenziale

La forma generale di un enum è questa:

enum NomeEnum {
    VALORE1,
    VALORE2,
    VALORE3
};

Esempio base

enum Livello {
    BASSO,
    MEDIO,
    ALTO
};

Livello difficolta = MEDIO;
Valori automatici

Se non specifichi nulla, i valori partono da 0 e aumentano di 1.

Valori espliciti

Puoi assegnare numeri precisi, per esempio ROSSO = 10.

Valori espliciti

enum Stato {
    SPENTO = 0,
    ACCESO = 1
};

4. Esempi guidati

Esempio 1 • Variabile con valori simbolici

#include <iostream>
using namespace std;

enum Semaforo {
    ROSSO,
    GIALLO,
    VERDE
};

int main(){
    Semaforo stato = VERDE;
}

In questo caso stato non contiene un numero scritto a mano, ma uno dei valori previsti dall'enum Semaforo.

Esempio 2 • Uso con if

#include <iostream>
using namespace std;

enum StatoOrdine {
    IN_ATTESA,
    SPEDITO,
    CONSEGNATO
};

int main(){
    StatoOrdine stato = SPEDITO;

    if(stato == SPEDITO){
        cout << "Ordine in viaggio" << endl;
    }
}

Il confronto è molto più leggibile rispetto a scrivere numeri come if(stato == 1).

Esempio 3 • Uso con switch

#include <iostream>
using namespace std;

enum Giorno {
    LUNEDI,
    MARTEDI,
    MERCOLEDI
};

int main(){
    Giorno oggi = MARTEDI;

    switch(oggi){
        case LUNEDI:
            cout << "Inizio settimana" << endl;
            break;
        case MARTEDI:
            cout << "Secondo giorno" << endl;
            break;
        case MERCOLEDI:
            cout << "Meta percorso" << endl;
            break;
    }
}

Esempio 4 • Caso realistico: priorità di ticket

enum Priorita {
    BASSA,
    MEDIA,
    ALTA,
    URGENTE
};
Qui l'enum è perfetto: le priorità possibili sono poche, ben definite e non devono essere inventate liberamente dall'utente.

L'enum migliora il codice soprattutto quando vuoi evitare numeri scritti "a caso" e rendere immediato il significato delle variabili.

6. Enum vs int

Molti studenti all'inizio si chiedono: perché usare un enum, se posso usare un normale int? La risposta è nella leggibilità.

Con int

int stato = 2;

Ma 2 cosa significa? Bisogna ricordarlo a memoria.

Con enum

StatoOrdine stato = CONSEGNATO;

Il significato è evidente già leggendo il codice.

L'enum non serve a fare “più cose” dell'int, ma a scrivere codice molto più chiaro e ordinato.

7. Errori tipici degli studenti

Errore 1 • Pensare che enum sia una lista libera di parole

No: i valori sono fissi e definiti dal programmatore.

Errore 2 • Confondere nome dell'enum e valori interni

Giorno è il tipo. LUNEDI, MARTEDI e simili sono i valori.

Errore 3 • Dimenticare il punto e virgola finale

enum Colore {
    ROSSO,
    VERDE,
    BLU
};

Errore 4 • Usare l'enum quando i valori non sono limitati

Se il dato può assumere infiniti numeri o testi diversi, l'enum non è adatto.

Errore 5 • Non capire che dietro ci sono numeri interi

Anche se usi nomi simbolici, ogni valore dell'enum corrisponde internamente a un intero.

10. Riepilogo finale

  • L'enum definisce un insieme limitato di valori simbolici.
  • Rende il codice più leggibile e meno ambiguo.
  • Dietro le quinte i valori corrispondono a interi.
  • Se non assegni numeri espliciti, la numerazione parte da 0.
  • È utile per stati, categorie, livelli, giorni, priorità e casi simili.
  • Aiuta a evitare i cosiddetti numeri “magici”.
Se uno studente capisce bene l'enum, inizia a scrivere codice più leggibile, più ordinato e più facile da mantenere.
